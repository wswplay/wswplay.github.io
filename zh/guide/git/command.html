# Mac 安装 Git

官网 (opens new window)。又是 brew，又是 Xcode，巴拉巴拉。。。
直接文件安装 (opens new window)，不香吗？
安装完成，git 一下，你就知道：会报错。

xcrun: error: invalid active developer path (/Library/Developer/CommandLineTools),
missing xcrun at: /Library/Developer/CommandLineTools/usr/bin/xcrun
1
2

Mac 个傻逼，git 还要安装 CommandLineTools，那下载安装命令行工具吧CommandLineTools (opens new window)。
安装完成，就大功告成！🎯

# 只 clone 某个文件夹

git config core.sparsecheckout true #设置允许克隆子目录
echo 'tt*' >> .git/info/sparse-checkout #设置要克隆的仓库的子目录路径，空格别漏
git remote add origin git@github.com:mygithub/test.git  #这里换成你要克隆的项目和库
git pull origin master    #下载

# 删除远程提交地址

# 本地项目与远程项目关联

# 本地项目内运行如下命令
# 查看远端信息
git remote -v
# 添加关联远程仓库
git remote add origin https://gitee.com/xxx/xx-xxx-xxx.git
# 再次查看，就能看到相关信息了
git remote -v
# 拉取代码
git pull origin master
# 如果出现如下错误(拒绝合并没有关联的历史记录)
fatal: refusing to merge unrelated histories
# 运行下面命令即可(即：允许合并没有关联的历史记录)
git pull origin master --allow-unrelated-histories

# 撤销 commit 提交

# 撤销最近一次commit
git reset --soft HEAD^
# 慎用！慎用！慎用！撤销commit，且会删除所有改动的代码。一夜回到解放前！
git reset --hard HEAD~1
1
2
3
4

提示

HEAD^ 表示上一个版本，即上一次的 commit，也可以写成 HEAD~1
如果进行两次的 commit，想要都撤回，可以使用 HEAD~2

# 修改 commit 注释

# 输入以下命令，会进入vim编辑器，修改完成保存即可
git commit --amend

# 项目内设置 git 对文件名 大小写敏感

默认为不敏感，当文件名只是更改大小写时，git 无法识别更新，导致报错。

# 查看忽略的设置
git config --get core.ignorecase
# 设置大小写敏感
git config core.ignorecase false
